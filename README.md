# 秘协作 (Secret Collab) - 隐私保护与智能协同深度融合的新型AI代理系统

## 1. 项目简介
本项目构建了一种隐私保护与智能协同深度融合的新型AI代理系统，其核心架构基于大语言模型（LLM）驱动的个性化代理、联邦学习（FL）框架、同态加密（CKKS）与安全多方计算（MPC）等前沿技术，形成“蜂群智能”协作体系。

项目面向隐私保护与智能协作并重的国家数字发展需求，在用户终端部署个性化 AI 代理，实现偏好与约束的本地学习，并通过同态加密、差分隐私等技术完成脱敏协作。云端在不接触明文数据的前提下开展联合推理与协同优化，支持会议调度与群体规划等场景，突破中心化智能系统隐私泄露与协同受限问题，为可信人工智能提供技术路径。

## 2. 系统架构

### 2.1 基础层：个性化大语言模型（LLM）驱动的数字孪生代理
- **代理架构**：每个用户拥有一个由大语言模型驱动的AI代理，作为其数字孪生体。
- **技术实现**：
  - **LoRA微调**：在用户本地设备上对通用LLM（如 Llama 3-8B）进行轻量化微调，适配用户个性化需求。
  - **增量学习**：每日仅需1-2次更新，避免频繁训练带来的性能损耗。
  - **多模态输入**：支持日历、文档、邮件等多源数据的语义解析与结构化存储。

### 2.2 核心层：联邦学习（FL）驱动的蜂群智能协作
- **联邦学习框架**：
  - **分布式协作机制**：当用户提出复杂任务（如“策划团队团建活动”），相关代理启动联邦学习流程。
  - **本地推理与微调**：仅交换任务相关的抽象信息（如偏好向量、思维链、计划片段），而非原始数据。
- **抽象信息交换**：
  - **思维链（Thought Chain）**：代理输出的逻辑推理路径，经加密后共享。
  - **偏好向量（Preference Vector）**：用户对时间、地点、预算等维度的偏好编码。

### 2.3 安全层：隐私保护与机密计算技术
- **同态加密（CKKS）**：支持浮点数运算，确保数据在加密状态下可直接计算，服务器无法解密原始数据。
- **差分隐私（DP）**：在梯度更新或聚合阶段，添加拉普拉斯/高斯噪声，满足(ε, δ)-差分隐私约束。
- **安全多方计算（MPC）**：使用混淆电路（Garbled Circuits）计算交集概率，避免暴露个体偏好。

### 2.4 应用层：多场景智能服务
- **会议时间协商**：代理自动分析用户日历，推荐最优时间。
- **旅游计划策划**：结合用户偏好（自然/文化/美食）与预算，生成个性化行程。
- **团队活动组织**：协调多人需求，输出高满意度方案。

## 3. 核心功能模块

### 3.1 协作中心 (Dashboard)
- **蜂群智能状态监控**：实时显示联邦节点连接数与在线状态。
- **待办事项智能分类**：基于意图识别自动分类会议邀请、行程方案与协作推荐。
- **隐私安全评分**：基于本地数据处理量与云端加密摘要比例计算的安全得分。

### 3.2 联邦调度 (Scheduler)
- **零隐私泄露时间调度**：通过 CKKS 加密的时间槽位交集计算。
- **智能冲突解决**：基于 LoRA 学习的用户偏好自动处理日程冲突。

### 3.3 跨域行程 (Travel)
- **多方隐私保护出行**：支持多人异地出发的行程智能规划。
- **加密位置服务**：基于差分隐私的位置模糊化查询。

### 3.4 智能推荐 (Recommendation)
- **联邦资源匹配**：基于群体偏好的协同过滤推荐。
- **知识蒸馏**：本地学生模型从全局教师模型获取最新推荐策略。

### 3.5 本地代理配置 (Agent Config)
- **自然语言交互训练**：通过对话形式训练本地代理的偏好与习惯。
- **实时技术监控**：可视化展示 LoRA Rank、Privacy Budget (ε) 及联邦聚合轮次。

### 3.6 安全效能分析 (Analytics)
- **信息熵监控**：实时计算数据混淆度（Entropy > 0.9）。
- **隐私泄露风险预警**：基于模型反演攻击模拟的风险评估。

## 4. 技术指标
- **隐私保护**：信息熵 > 0.9，满足(ε, δ)-差分隐私约束。
- **协作效率**：本地模型推理延迟 < 50ms (NPU加速)。
- **模型精度**：个性化推荐匹配率 > 92%。
- **通信开销**：相比原始数据传输降低 95% 以上。

## 5. 运行环境与部署说明

### 5.1 前端环境
- **框架**：React 18 + TypeScript + Vite
- **UI 组件库**：Tailwind CSS + Lucide React
- **图表库**：Recharts
- **启动命令**：`npm run dev`

### 5.2 后端环境 (MVP版本)
- **运行时**：Node.js v16+
- **框架**：Express.js + Socket.IO
- **功能**：
  - RESTful API：提供用户鉴权与元数据查询
  - WebSocket：实现联邦节点实时协调与模型参数分发
- **启动命令**：`node backend/server.js`

### 5.3 局域网联调
1. 启动后端服务：在终端执行 `node backend/server.js`（默认端口 3001）。
2. 启动前端服务：在终端执行 `npm run dev -- --host`（默认端口 3000）。
3. 访问地址：在同一局域网设备输入前端显示的 IP 地址（如 `http://192.168.1.5:3000`）。
